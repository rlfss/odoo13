Website RMA (Return Merchandise Authorization) in Odoo

With the Website RMA in Odoo, you enable your customer to place the product return request from your web store, created with Odoo. It will generate the RMA number when a customer initiates the exchange or return the product you sold. The request will show up at Odoo backend and appropriate action can be taken by admin accordingly. Consequently, the state of progress can be tracked by the customer from its account.

Is this app compatible with Odoo Enterprise as well as Community?

Yes, the app works perfectly well with Odoo Enterprise (On-premise and Odoo.SH) as well as Community. Odoo Online (Cloud) does not allow installation of third-party apps and hence this app cannot be installed on Odoo Online.

You can contact us to request a customization in the app.

How can a customer make a return request on my Odoo website?

After purchasing any products from your Odoo Website, a ‘Return Order’ button will appear on the Sales Order portal through which a customer can make a return request.

What is the required information that customer should fill while requesting the RMA in Odoo in Website?

While requesting RMA on Odoo Website, a customer should fill the Return Reason, Quantity to Return and optionally enter Return Note if necessary.

What is the RMA Number that is given to the customer?

RMA Number is the Unique ID generated on Odoo Website after the RMA Request is successfully placed by the customer. It is used to track the RMA Request.

How a customer can track the RMA Orders from Odoo Website?

After logging-in to Odoo Website, a customer can track its RMA Orders from My Account section.

Who can authorize the RMA Orders placed by the customer?

Only an admin level user can authorize the RMA Orders, not the normal user. However, a normal user can download the RMA Orders for various purposes, if required.
